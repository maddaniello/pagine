Osservazioni e raccomandazioni
01
Dismettere o documentare gli account vuoti. Gli account GTM 6080855706 e 6266506288 contengono container senza tag pubblicati (4 versioni e 1 versione rispettivamente). Probabilmente erano destinati a configurazioni native di Google Ads e GA4 poi confluite nel container principale. Verificare se i tag ID associati (AW-11093796799, G-VLWGDJ3VZN) sono ancora attivi a livello di account Google e, in caso, decidere se dismettere i container o documentarne lo scopo.
02
Pulire i tag Google Ads in pausa nel container web. I 9 tag GADS in pausa (Conversion Linker, ottieni_mutuo, StartFlow, Lead Inviata, calcola_rata, Visualizzazione_Consulenza, Remarketing, User Data Global, learning display, Visualizzazione-rata, Visualizzazione Contattaci) occupano spazio nel container e possono generare confusione. Se il tracking GAds è interamente migrato al server-side, questi tag possono essere eliminati.
03
Workspace "Esercitazione formazione" nel container di produzione. Il workspace secondario (id: 67) nel container GTM-DRHC5QQ potrebbe contenere modifiche non pubblicate risalenti all'attività di formazione. Verificarne il contenuto e, se non necessario, eliminarlo per evitare confusione durante future attività di manutenzione.
04
Trigger duplicato "Copy of trigger abbandono-flusso". Il trigger id 292 (historyChange) sembra un duplicato creato per test del trigger id 281 (customEvent). Verificare se entrambi sono in uso o se il "Copy of" è un residuo da eliminare.
05
Tag di test in pausa nel container server. I tag "GA4 - test property" (id: 153), "GA4 - Test - Offline events" (id: 175) e "Stape Writer" (id: 148) risultano in pausa. Se il collaudo è concluso, valutare l'eliminazione per mantenere il container server pulito.
06
Architettura client-server ben strutturata. L'implementazione con container web (GTM-DRHC5QQ) che invia eventi al container server (GTM-TBP9CXC) per la gestione delle Conversion API di Facebook, Google Ads offline e TikTok è corretta e segue le best practice. Il cookie monster è correttamente in posizione server-side, così come l'hashing dei dati PII (email, telefono, external_id).
07
Consent Mode v2 implementato correttamente. Il container web gestisce il consenso tramite Cookiebot con tag Default e Update di Google Consent Mode v2. Le blocking condition sui trigger Facebook, Google Ads e Hotjar sono presenti e coerenti. Il container server integra il segnale di consenso attraverso la variabile "cookie - CookieConsent" e il filtro "cookiebot user agent".
